언리얼 디버깅 TIL: GetName()과 Actor Label 차이

맵에 배치된 CineCameraActor들을 태그로 수집한 뒤 이름 기준으로 정렬했지만, 예상과 다른 카메라가 선택되는 문제를 디버깅한 기록이다.

상황

맵에는 CineCameraActor가 3개 배치되어 있고, 각각 같은 태그가 지정되어 있다.

코드에서는 맵에 있는 Actor 중 LobbyMainCamera 태그가 붙은 Actor를 수집해서 FoundCameras 배열에 넣고 있었다.

UGameplayStatics::GetAllActorsWithTag(
    GetWorld(),
    TEXT("LobbyMainCamera"),
    FoundCameras
);

FoundCameras.Sort([](const AActor& A, const AActor& B)
{
    return A.GetName() < B.GetName();
});

필자는 배열이 다음 순서로 정렬될 것이라고 예상했다.

CineCameraActor
CineCameraActor2
CineCameraActor3

그래서 배열의 첫 번째 액터에 접근하면 SECTION 1을 비추는 카메라가 선택될 것이라고 생각했다.

문제

예상과 달리 SECTION 1을 보여줘야 할 카메라가 SECTION 2를 보여주고 있었다.

즉, 배열의 첫 번째 요소가 내가 생각한 CineCameraActor가 아니었다.

디버깅 과정

1. 배열이 정상적으로 수집되었는지 확인

먼저 FoundCameras 배열에 카메라들이 전부 들어오는지 확인했다.

태그 설정은 정상적이었고, 관련 Actor들도 정상적으로 수집되고 있었다.

2. Actor 정보를 펼쳐보기

디버깅 중 Actor 정보를 펼쳐보니 원인의 실마리가 보였다.

Actor에는 Label 이름과 Name 이름이 따로 존재했다.

핵심: 뷰포트나 월드 아웃라이너에서 보이는 이름은 GetName()의 이름이 아니다.

뷰포트에서 보이는 이름은 Actor Label이다.

3. 정렬 기준 확인

기존 코드는 GetName()을 기준으로 정렬하고 있었다.

그래서 월드 아웃라이너에서 보이는 순서와 실제 정렬 결과가 달라졌다.

원인

언리얼에는 Actor 이름과 관련해서 크게 두 가지 개념이 있다.

구분 의미 사용처
GetName() UObject의 내부 Object Name 객체 식별, 디버깅, 내부 참조
GetActorLabel() 에디터에서 보이는 Actor Label 월드 아웃라이너 표시 이름

예를 들어 월드 아웃라이너에는 다음처럼 보일 수 있다.

CineCameraActor
CineCameraActor2
CineCameraActor3

하지만 실제 GetName() 값은 다음처럼 다를 수 있다.

CineCameraActor_1
CineCameraActor_0
CineCameraActor_3

즉, Label 이름만 보고 정렬 결과를 예상하면 안 된다.

해결

에디터에서 보이는 이름 기준으로 정렬하고 싶다면 GetName() 대신 GetActorLabel()을 사용하면 된다.

FoundCameras.Sort([](const AActor& A, const AActor& B)
{
    return A.GetActorLabel() < B.GetActorLabel();
});

이렇게 수정하자 원하는 SECTION 1 구역을 비추는 카메라에 정상적으로 접근할 수 있었다.

번외: GetName()은 언제 정해질까?

GetName()은 UObject가 생성될 때 정해지는 내부 이름이다.

CineCameraActor를 Place Actors 패널에서 드래그 앤 드롭해서 월드에 배치하면, 객체가 맵에 인스턴스화되는 순간 내부 Object Name이 정해진다.

그리고 같은 이름이 이미 존재하면 언리얼이 자동으로 고유한 숫자를 붙인다.

테스트 결과, Label 이름은 같아도 새로 만들거나 삭제 후 다시 만들면 GetName()의 숫자가 달라지는 것을 확인할 수 있었다.

즉, GetName()은 사용자가 보는 이름이라기보다 객체가 생성될 때 정해지는 내부 식별 이름에 가깝다.

정리

  • 뷰포트와 월드 아웃라이너에 보이는 이름은 Actor Label이다.
  • GetName()은 UObject의 내부 Object Name이다.
  • Label과 Name은 서로 다를 수 있다.
  • 에디터 표시 이름 기준으로 정렬하려면 GetActorLabel()을 사용한다.
  • 게임 로직을 GetName()에 의존해서 작성하는 것은 좋지 않다.

현업에서 더 권장되는 방식

카메라 순서처럼 게임 로직에 필요한 순서는 이름이 아니라 명시적인 데이터로 관리하는 편이 좋다.

  • UPROPERTY(EditAnywhere) int32 Order
  • Gameplay Tag
  • Enum
  • Data Asset
  • 배열에 직접 등록
UPROPERTY(EditAnywhere)
int32 CameraOrder = 0;

FoundCameras.Sort([](const AMyCameraActor& A, const AMyCameraActor& B)
{
    return A.CameraOrder < B.CameraOrder;
});

이렇게 하면 에디터 이름 변경이나 객체 생성 순서와 상관없이 안정적으로 원하는 순서를 보장할 수 있다.
